## Формулировка задания вариативной части. Этот файл служит точкой входа для всех, кто работает с репозиторием

Разработать игровой продукт на языке Python с применением библиотеки PyGame. 

1. **Игровое поле** – сетка 12×25 клеток, корректное отображение границ и занятых ячеек.
2. **Тетромино** – 7 классических фигур с текстурами, реализованы:
   - падение с гравитацией (привязано ко времени);
   - перемещение влево/вправо с настраиваемой задержкой;
   - мгновенный сброс (hard drop);
   - вращение с учётом стен и коллизий.
3. **Очистка линий и комбо** – заполненные горизонтальные линии исчезают, блоки сверху смещаются. Система комбо: очки зависят от количества линий, уровня и множителя комбо.
4. **Конец игры** – если новая фигура не может появиться, игра останавливается, выводится «Game Over», финальный счёт и запас маны.
5. **Система маны** – мана пополняется при приземлении фигуры и очистке линий. Максимум – 100 единиц.
6. **Три базовые способности** (активируются клавишами `1`, `2`, `3`):
   - **Замена** (55 маны) – заменяет текущую фигуру на удерживаемую, если та существует.
   - **Замедление** (70 маны) – замедляет скорость падения в 3 раза на 4 секунды.
   - **Пропуск** (40 маны) – мгновенно сбрасывает текущую фигуру вниз до упора.
7. **Управление** – поддержка клавиатуры (стрелки, пробел для hard drop, `C` для hold, `P` для паузы, `M` для музыки, `R` для рестарта при Game Over, `ESC` для выхода в меню).
8. **Стабильность** – игра не «падает» при быстром вводе, выдерживает 60 FPS.
